**Step 2: Create `src/pipeline/score.py`** implementing scoring per D-04, D-05, D-06, D-07, SCOR-01 through SCOR-04.

The module must have one public function: `score_and_filter(prospects: list[ProspectDict], skip_list_path: str = "skip_list.json") -> list[ProspectDict]`

Implementation details for `score_and_filter`:
- Load skip list from `skip_list_path` using `json.load()`. Build two sets: `skip_domains` (lowered) and `skip_names` (lowered)
- For each prospect in the list, process in this order:

**A) Skip list check (per D-06, SCOR-03):**
- If prospect domain (lowered) is in skip_domains OR prospect company_name (lowered) is in skip_names:
  - Set `status = "skipped"`, `score = 0`
  - Continue to next prospect

**B) Hard filters (per D-05, SCOR-01):**
- If `enrichment_status == "enrichment-failed"`: set `status = "disqualified"`, `disqualify_reason = "enrichment_failed"`, `score = 0`
- If `company_size is None`: set `status = "disqualified"`, `disqualify_reason = "missing_company_size"`, `score = 0` (D-03: solo creators dropped here silently)
- If `company_size < 50 or company_size > 2000`: set `status = "disqualified"`, `disqualify_reason = "company_size_out_of_range"`, `score = 0`
- If `cadence_days is None or cadence_days > 90`: set `status = "disqualified"`, `disqualify_reason = "inactive_or_infrequent_show"`, `score = 0`
- If `work_email` is falsy (None or empty string): set `status = "disqualified"`, `disqualify_reason = "no_contact_info"`, `score = 0`

**C) Weighted scoring (per D-04, SCOR-02):**
Implement these four sub-functions:

`size_fit_score(size: int) -> float` (max 40 points):
- 100 <= size <= 1000: return 40 (perfect ICP range)
- 50 <= size < 100 or 1000 < size <= 2000: return 20 (edge of ICP)
- else: return 0

`cadence_fit_score(cadence_days: float) -> float` (max 30 points):
- cadence_days <= 7: return 30 (weekly)
- cadence_days <= 16: return 25 (biweekly)
- cadence_days <= 35: return 15 (monthly)
- else: return 5

`data_completeness_score(prospect: ProspectDict) -> float` (max 20 points):
- Check these 5 fields: work_email, host_name, company_name, company_size, recent_episode_topic
- Score = round((present_count / 5) * 20)

`category_match_score(prospect: ProspectDict) -> float` (max 10 points):
- B2B categories: {"Business", "Technology", "Marketing", "Entrepreneurship", "Management"}
- If any value in prospect.get("categories", {}) is in b2b_cats: return 10
- Else: return 0

Total score = size_fit + cadence_fit + data_completeness + category_match (0-100 scale)
- Set `prospect["score"] = round(total)`
- If total >= 60: set `status = "call-ready"` (per D-07)
- Else: set `status = "below-threshold"` (per D-07)

**D) Cheat sheet assembly (per ENRI-02, D-11):**
After scoring, for every prospect (regardless of status), call `assemble_cheat_sheet(prospect)` which creates a JSON string containing:

```python
{

        "recent_episode_topic": prospect.get("recent_episode_topic"),
        "episode_count": prospect.get("episode_count")
    },
    "scoring": {
        "score": prospect.get("score"),
        "status": prospect.get("status")
    }
}
```
Set `prospect["cheat_sheet"] = json.dumps(cheat_sheet_data, indent=2)`
