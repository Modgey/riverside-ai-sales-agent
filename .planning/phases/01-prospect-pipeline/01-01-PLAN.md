**Step 6: Create `src/pipeline/discover.py`** implementing discovery per D-01, DISC-01, DISC-02, DISC-03:

The module must have one public function: `discover_prospects(keywords=None, max_per_term=50) -> list[ProspectDict]`

Implementation details:
- Load env vars with `dotenv.load_dotenv()` using `find_dotenv()` to locate `.env` in `src/` directory
- Initialize podcastindex with `podcastindex.init({"api_key": ..., "api_secret": ...})`
- Default search terms: `["B2B SaaS podcast", "marketing podcast", "business technology podcast", "demand generation podcast", "revenue growth podcast", "startup podcast"]` (per D-01 category-first strategy, but Podcast Index search is term-based per Pitfall 3)
- For each term, call `index.search(term, clean=True)`, collect feeds from `results.get("feeds", [])`
- Deduplicate on feed URL using a `seen` set
- Apply recency gate: filter out feeds where `lastPublishTime` is older than 90 days (per Pitfall 5). Use `time.time() - (90 * 24 * 60 * 60)` as cutoff
- For each surviving feed, call a `parse_rss(rss_url)` function that uses feedparser to extract:
  - `host_name`: use `feed.feed.get("itunes_author") or feed.feed.get("author") or feed.feed.get("title", "Unknown")` (per Pitfall 4, prefer itunes_author)
  - `podcast_name`: `feed.feed.get("title", "")`
  - `cadence_days`: compute from date deltas of last 10 episodes' `published_parsed`. If fewer than 2 episodes with dates, set to `None`
  - `recent_episode_topic`: `entries[0].get("title", "")` if entries exist
  - `episode_count`: `len(feed.entries)`
  - `domain`: extract domain from feed link or feed URL using urllib.parse
- Wrap RSS parsing in try/except to handle malformed feeds (log warning, skip feed)
- Print structured progress per D-16: `f"Searching '{term}'... found {count} feeds"` for each term, then `f"Discovery complete: {len(unique)} unique feeds, {len(active)} active, {len(parsed)} parsed successfully"`
- Return list of ProspectDict with fields: podcast_name, host_name, rss_url, domain, cadence_days, recent_episode_topic, episode_count, categories, last_publish_time
- Add `time.sleep(0.3)` between RSS fetches to be polite to feed servers

Include a `if __name__ == "__main__"` block that calls `discover_prospects()` and prints the count and first 3 records for smoke testing.

Create `src/pipeline/enrich.py` implementing Apollo enrichment per ENRI-01 and D-15.

The module must have one public function: `enrich_prospects(prospects: list[ProspectDict]) -> list[ProspectDict]`

Implementation details:
- Load env vars with `dotenv.load_dotenv()` using `find_dotenv()`
- Get `APOLLO_API_KEY` from `os.getenv("APOLLO_API_KEY")`
- Define `APOLLO_BASE = "https://api.apollo.io/api/v1"`

Helper function `enrich_person(host_name: str, domain: str, api_key: str) -> dict | None`:
- POST to `f"{APOLLO_BASE}/people/match"` with headers `{"x-api-key": api_key, "Content-Type": "application/json"}` and json `{"name": host_name, "domain": domain, "reveal_personal_emails": False, "reveal_phone_number": False}`
- timeout=10
- If status_code == 422, return None (credits exhausted per Pitfall 2)
- If status_code != 200, return None
- Extract from `resp.json().get("person", {})`: work_email (key: "email"), title (key: "title"), company_size from `organization.estimated_num_employees`, industry from `organization.industry`, company_name from `organization.name`
- Return dict with those fields

Helper function `enrich_org(domain: str, api_key: str) -> dict`:
- POST to `f"{APOLLO_BASE}/organizations/enrich"` with same headers, json `{"domain": domain}`, timeout=10
- If status != 200, return empty dict
- Extract from `resp.json().get("organization", {})`: company_size (estimated_num_employees), industry, company_name (name)
- Return dict with those fields

Main function `enrich_prospects`:
- Iterate over prospects list
- For each prospect, call `enrich_person(prospect["host_name"], prospect["domain"], api_key)`
- If person enrichment returns None or has no email, try `enrich_org(prospect["domain"], api_key)` as fallback for company data
- If both return no useful data (no email AND no company_size), set `prospect["enrichment_status"] = "enrichment-failed"` and print warning: `f"Warning: enrichment failed for {prospect['podcast_name']} ({prospect['domain']})"`  (per D-15)
- If enrichment succeeds, merge the enriched fields into the prospect dict and set `prospect["enrichment_status"] = "enriched"`
- Add `time.sleep(0.5)` between Apollo calls to avoid 429 rate limits (per anti-pattern in RESEARCH.md)
- Print structured progress per D-16: every 10 prospects print `f"Enriching... {i}/{total} complete"`, and at the end print `f"Enrichment complete: {enriched_count} enriched, {failed_count} failed"`
- Return the full list (including failed ones, so scoring can handle them)

Wrap all requests in try/except for `requests.RequestException`, returning None on failure.

Include `if __name__ == "__main__"` block that creates 2 sample ProspectDicts with known domains (e.g., "hubspot.com", "drift.com") and runs enrichment for smoke testing.
