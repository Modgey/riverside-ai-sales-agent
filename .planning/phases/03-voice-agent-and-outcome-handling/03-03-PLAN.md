**File 1: src/tool_server.py** — Minimal Flask server for mid-call tool webhooks (per D-13, simulated)

```python
"""Simulated tool webhook server for Vapi mid-call function calls.

Run this before placing calls: py src/tool_server.py
Then expose via ngrok: ngrok http 3000
Set TOOL_SERVER_URL in .env to the ngrok URL + /tools
"""

from flask import Flask, request, jsonify

app = Flask(__name__)

TOOL_RESPONSES = {
    "book_meeting": "Great, I've sent a calendar invite to schedule a demo. They should see it in their inbox shortly.",
    "send_signup_link": "Done, I've sent them a link to sign up for a free Riverside trial. They can get started right away.",
}


@app.route("/tools", methods=["POST"])
def handle_tool_call():
    data = request.json
    tool_calls = data.get("message", {}).get("toolCallList", [])

    results = []
    for tc in tool_calls:
        func_name = tc.get("function", {}).get("name", "")
        args = tc.get("function", {}).get("arguments", {})
        response_text = TOOL_RESPONSES.get(func_name, "Action completed.")
        print(f"  Tool call: {func_name}({args}) -> {response_text[:50]}...")
        results.append({
            "toolCallId": tc["id"],
            "result": response_text,
        })

    return jsonify({"results": results})


@app.route("/health", methods=["GET"])
def health():
    return jsonify({"status": "ok", "tools": list(TOOL_RESPONSES.keys())})


if __name__ == "__main__":
    print("Tool server running on http://localhost:3000")
    print("Endpoints:")
    print("  POST /tools  - Handle Vapi tool calls (book_meeting, send_signup_link)")
    print("  GET  /health - Health check")
    print()
    print("To expose publicly: ngrok http 3000")
    print("Then set TOOL_SERVER_URL=<ngrok-url>/tools in .env")
    app.run(port=3000, debug=False)
```

**Functions to implement:**

1. `load_call_ready_prospects() -> list[dict]`:
   - Load from `data/call_context.json`.
   - Filter to prospects with `status == "Qualified"` AND `call_context` is not None/empty.
   - Return the filtered list.

2. `find_prospect(prospects: list[dict], name: str) -> dict | None`:
   - Search by podcast_name (case-insensitive substring match).
   - Return first match or None.

3. `run_call(prospect: dict, target_number: str) -> dict`:
   - Print header: `"\n{'='*60}\nCALLING: {prospect['podcast_name']}\n{'='*60}"`
   - Print prospect summary: name, company, title.
   - Call `place_call(prospect, target_number)` to initiate the call.
   - Extract call_id from the returned call object.
   - Print: `"  Call initiated: {call_id}"`.
   - Print: `"  Waiting for call to complete (this may take several minutes)..."`.
   - Initialize Vapi client: `Vapi(token=os.getenv("VAPI_API_KEY"))`.
   - Call `process_call_outcome(client, call_id, prospect)` to poll, classify, and write to Airtable.
   - Return the updated prospect dict.

4. `run_all_calls(prospects: list[dict], target_number: str) -> list[dict]`:
   - Iterate through prospects sequentially (one call at a time).
   - Call `run_call()` for each.
   - After each call, save results to `data/call_results.json`.
   - Print summary at end: total calls, outcomes breakdown.
   - Return list of updated prospect dicts.

5. `list_prospects(prospects: list[dict])`:
   - Print a table of call-ready prospects: index, podcast_name, host_name, company_name, has call_context (yes/no).
   - Used for `--list` mode.

6. CLI parsing with argparse:
   - `--prospect NAME`: call a specific prospect by podcast name
   - `--all`: call all qualified prospects with call context
   - `--number NUMBER`: target phone number (required for --prospect and --all)
   - `--list`: show available prospects
   - `--dry-run`: build and print the assistant config without placing a call (for testing)

7. `if __name__ == "__main__"` block:
   - Parse args.
   - If `--list`: load prospects, call list_prospects, exit.
   - If `--dry-run`: print `build_assistant_config()` as JSON, exit.
   - If `--prospect`: load prospects, find by name, call run_call with --number.
   - If `--all`: load prospects, call run_all_calls with --number.
   - Save final results to `data/call_results.json`.

IMPORTANT per VOIC-07 / D-12: The call runner must NOT modify the assistant config between calls. Only `assistant_overrides.variable_values` changes per prospect. The `build_assistant_config()` call happens once and is reused. Actually, since place_call() calls build_assistant_config() internally, that's fine -- it rebuilds from the same harness.txt each time. The harness text never changes.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Vapi platform -> tool_server.py | Vapi POSTs tool invocations to our webhook. Untrusted inbound HTTP. |
| call_runner.py -> Vapi API | Sends prospect data and call config to external service |
| tool_server.py -> public internet | ngrok exposes local server to any incoming HTTP |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Spoofing | tool_server.py | accept | Demo-only server with canned responses. No real actions taken. No auth needed because responses are static and harmless. Production would validate Vapi webhook signatures. |
| T-03-11 | Information Disclosure | tool_server.py via ngrok | mitigate | ngrok URL is ephemeral (changes each restart). Server returns only canned text, no prospect data. Shut down ngrok after demo. |
| T-03-12 | Tampering | tool_server.py POST body | accept | Server reads function name from POST body but only returns predefined canned responses. Malformed input returns "Action completed." default. No state mutation. |
| T-03-13 | Information Disclosure | call_runner.py | mitigate | Prospect phone numbers passed as CLI args, not stored in code. call_results.json stored locally in data/ dir (gitignored). |
| T-03-14 | Elevation of Privilege | tool_server.py | accept | Server runs on localhost:3000, only exposed via ngrok during active demo. No file system access, no database access, no auth tokens. |
</threat_model>
