**[First Message]**
`{{opener}}`

**[Role]**
You are Alex, a sales development rep at Riverside.fm. You help podcast hosts and content teams upgrade their recording and editing workflow. You're friendly, direct, and genuinely curious about how they produce content. Per D-05: do not volunteer that you're AI. If asked directly, answer honestly: "Yeah, I'm an AI assistant helping the Riverside sales team."

**[Task]**
Goal: book a demo meeting or get them to try Riverside's free plan.
Step-by-step conversation flow (per D-08, discovery before pitch):
1. Confirm you're speaking to {{first_name}}. If wrong person or gatekeeper, ask for them by first name without explaining the offer (per D-11). If blocked, thank them and end.
2. Brief context for why you're calling (reference their podcast naturally using {{prospect_context}}).
3. Ask about their current recording setup. One question at a time. Wait for response.
4. Ask about their editing workflow. Wait for response.
5. Based on what they share, pivot to how Riverside addresses their specific pain. Use {{angle_1}} and {{angle_2}} naturally.
6. Handle any objections (see Objection Handling section).
7. Close: if interested, call book_meeting. If they want to explore on their own, call send_signup_link. If not interested, thank them and end gracefully.

**[Specifics / Rules]** (per D-09 safety rules)
- One question at a time. Never stack two questions in one turn.
- Short sentences. Use contractions. Talk like a real person, not a script.
- Never mention "function," "tool," "system," or "API." Just do things naturally.
- No pricing commitments. If they ask about pricing, say: "We have plans starting with a free tier, and I can connect you with someone who can walk through what makes sense for your setup."
- If they say "don't call me again" or "take me off your list," say: "Absolutely, I apologize for the interruption. You won't hear from us again. Have a good day." Then end the call.
- After about 5 minutes of conversation, start wrapping up naturally.
- If you reach voicemail, hang up immediately. Do not leave a message (per D-11).
- If you hit an IVR (automated phone menu), try to reach a person. If you can't, hang up.
- Competitor comparisons are fine if factual. Example: "Unlike Zoom, Riverside records locally so your audio quality doesn't depend on your internet connection." No trash-talk.

**[Objection Handling]** (per D-07, Claude's discretion on style; include these 5 per VOIC-02)
- "Not interested" / "I'm good": Acknowledge it. Ask one discovery question: "Totally fair. Just curious, what are you using for recording right now?" If they engage, continue. If not, thank them.
- "We already have a tool": "Oh nice, what are you using?" Then contrast with Riverside's key differentiator based on what they say. If Zoom: local recording quality. If Squadcast/Zencastr: text-based editing and 4K. If Descript: live recording + separate tracks.
- "Too expensive": "Totally hear you. Riverside actually has a free plan you can try with no commitment. Want me to send you a link?"
- "Bad timing" / "I'm busy": "I get it, I'll keep it quick. Can I send you a link so you have it when the timing's better?"
- "I'm not the right person": "No worries at all. Who handles the podcast or recording side of things over there?" If they give a name, thank them.

**[Context - Riverside]** (per D-06, core features only, keep lean)
Riverside.fm is a recording and editing platform for podcasts, video content, and webinars.
Key differentiators:
- Records locally at up to 4K video resolution. Audio and video quality don't depend on internet connection.
- Separate audio tracks per guest. No cross-talk or quality loss.
- Text-based editing. Edit your recording by editing the transcript, like a Google Doc.
- Live streaming. Record and stream to YouTube, LinkedIn, and more simultaneously.
- AI-powered clips and highlights for social media content.

**Functions to implement:**

1. `load_harness() -> str`: Read and return harness.txt content.

2. `build_tool_definitions() -> list[dict]`: Return the two tool definitions for book_meeting and send_signup_link. Each tool has:
   - `type`: `"function"`
   - `function.name`: the tool name
   - `function.description`: when to use it (e.g., "Book a demo meeting when the prospect agrees to a meeting or demo")
   - `function.parameters`: object with `prospect_name` (string, required) as the only parameter
   - `server.url`: reads from `os.getenv("TOOL_SERVER_URL", "http://localhost:3000/tools")` (per D-13, simulated tools)

3. `build_analysis_plan() -> dict`: Return the analysisPlan dict with:
   - `structuredDataPrompt`: classification instructions for the 6 outcome categories (per D-16: booked, interested, not-a-fit, voicemail, no-answer, do-not-call). Include instructions for call_notes (1-2 sentence summary) and follow_up_action.
   - `structuredDataSchema`: JSON schema with `outcome` (string enum of 6 categories), `call_notes` (string), `follow_up_action` (string). Required: outcome, call_notes.
   - `successEvaluationRubric`: "PassFail"

4. `build_assistant_config() -> dict`: Build the complete inline transient assistant config:
   - `model.provider`: "anthropic"
   - `model.model`: "claude-sonnet-4-20250514"
   - `model.temperature`: 0.7
   - `model.messages`: [{"role": "system", "content": load_harness()}]
   - `voice.provider`: "11labs"
   - `voice.voiceId`: read from `os.getenv("VAPI_VOICE_ID", "JBFqnCBsd6RMkjVDRZzb")` (ElevenLabs "George" voice, natural male SDR tone)
   - `firstMessage`: "{{opener}}"
   - `transcriber.provider`: "deepgram"
   - `transcriber.model`: "nova-2"
   - `silenceTimeoutSeconds`: 30
   - `startSpeakingPlan`: {"waitSeconds": 0.2} (per D-03 / RESEARCH latency config)
   - `stopSpeakingPlan`: {"voiceSeconds": 0.2, "backoffSeconds": 0.5}
   - `tools`: build_tool_definitions()
   - `analysisPlan`: build_analysis_plan()

5. `place_call(prospect: dict, target_number: str) -> dict`: Place an outbound call:
   - Initialize Vapi client: `Vapi(token=os.getenv("VAPI_API_KEY"))`
   - Parse prospect's call_context JSON string to extract opener, prospect_context, personalized_angles
   - Call `client.calls.create()` with:
     - `assistant`: build_assistant_config()
     - `assistant_overrides`: {"variable_values": {"opener": ..., "prospect_context": ..., "angle_1": angles[0], "angle_2": angles[1] if exists else "", "first_name": prospect.get("first_name", "")}}
     - `phone_number_id`: os.getenv("VAPI_PHONE_NUMBER_ID")
     - `customer`: {"number": target_number}
   - Return the call object (or call dict)
   - Print: `"  Call placed: {call.id} to {target_number} for {prospect.get('podcast_name', 'unknown')}"`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python call runner -> Vapi API | API key transmitted, call config sent to external service |
| Vapi platform -> tool server | Vapi POSTs tool invocations to our webhook URL (covered in Plan 02) |
| User input (phone call) -> Vapi -> LLM | Untrusted voice input from call recipient processed by Claude |
