<objective>
Build the outcome handling pipeline: poll Vapi for post-call analysis results, extract the structured outcome classification, write results back to the prospect's Airtable row, and log follow-up actions. This module operates independently of the call runner (takes a call_id and prospect dict as input).

Purpose: After a call ends, Vapi's analysisPlan produces structured data (outcome category, notes, follow-up). This module retrieves that data and persists it to Airtable so the reviewer sees the complete prospect lifecycle in one row.
Output: outcome_handler.py (poll + extract + write), updated upload.py (field mappings), updated .env.example (Vapi keys).

```python
"""Post-call outcome handling: poll Vapi, classify, write to Airtable."""

import os
import time
from datetime import datetime, timezone

from dotenv import load_dotenv
from pyairtable import Api
from vapi import Vapi

load_dotenv(os.path.join(os.path.dirname(os.path.dirname(__file__)), ".env"))

VALID_OUTCOMES = {"booked", "interested", "not-a-fit", "voicemail", "no-answer", "do-not-call"}

FOLLOW_UP_ACTIONS = {
    "booked": "Send Calendly confirmation email to {email}",
    "interested": "Send Riverside free trial signup link to {email}",
    "not-a-fit": "No follow-up needed",
    "voicemail": "Schedule retry call in 48 hours",
    "no-answer": "Schedule retry call in 24 hours",
    "do-not-call": "Add to suppression list, no further contact",
}
```

**Functions to implement:**

1. `wait_for_call_result(client: Vapi, call_id: str, timeout: int = 600, poll_interval: int = 5) -> dict`:
   - Poll `client.calls.get(call_id)` every `poll_interval` seconds.
   - When `call.status == "ended"`: wait 5 more seconds for analysis to complete (per Pitfall 5), then re-poll.
   - If `call.analysis` and `call.analysis.structured_data` exist, return the call object.
   - If analysis is still None after 30 seconds of post-end polling, return the call anyway (graceful degradation).
   - Raise `TimeoutError` if call doesn't end within `timeout` seconds.
   - Print progress: `"  Waiting for call {call_id} to complete... ({elapsed}s)"` every 3rd poll.

2. `extract_outcome(call_result) -> dict`:
   - Extract from `call_result.analysis.structured_data` if available.
   - Return dict with keys: `outcome` (str), `call_notes` (str), `follow_up_action` (str).
   - If structured_data is None or missing fields, return defaults: `outcome="unknown"`, `call_notes="Analysis not available"`, `follow_up_action=""`.
   - Validate outcome is in VALID_OUTCOMES, default to "unknown" if not.

3. `log_follow_up(prospect: dict, outcome: str)`:
   - Look up the follow-up action from FOLLOW_UP_ACTIONS dict.
   - Format with prospect's email: `action.format(email=prospect.get("work_email", "no-email"))`.
   - Print to console: `"  Follow-up [{outcome}]: {formatted_action}"` (per D-18, logged not sent).
   - Return the formatted action string.

4. `update_prospect_outcome(prospect: dict, outcome_data: dict)`:
   - Connect to Airtable using same env vars as upload.py: `AIRTABLE_API_KEY`, `AIRTABLE_BASE_ID`, `AIRTABLE_PROSPECTS_TABLE_ID`.
   - Build fields dict: `podcast_name` (for upsert key), `call_outcome`, `call_notes`, `call_timestamp` (ISO 8601 UTC now), `follow_up_action`, `status` set to "Called".
   - Use `table.batch_upsert([{"fields": fields}], key_fields=["podcast_name"])`.
   - Print: `"  Airtable updated: {prospect['podcast_name']} -> {outcome}"`.

5. `process_call_outcome(client: Vapi, call_id: str, prospect: dict) -> dict`:
   - Orchestrator function that calls wait_for_call_result, extract_outcome, log_follow_up, update_prospect_outcome in sequence.
   - Returns the prospect dict updated with call_outcome, call_notes, call_timestamp, follow_up_action, call_id fields.
   - Wraps in try/except with graceful degradation: if any step fails, log the error and still return the prospect with whatever data was collected.

Add `if __name__ == "__main__"` block that prints the FOLLOW_UP_ACTIONS table for inspection.
