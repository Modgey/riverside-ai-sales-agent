```python
class CallContextResponse(BaseModel):

    riverside_hooks: list[str]
    narrative_briefing: str
```

**`call_llm_context(prospect: ProspectDict) -> CallContextResponse | None`:**
- Build user message as JSON with these fields extracted from ProspectDict:
  - `podcast_name`, `host_name`, `first_name`
  - `title` (role -- per D-12, this drives pitch framing)
  - `company_name`, `company_size`, `industry`
  - `recent_episode_topic`
  - `episode_details` (list of dicts with title, description, guest_name)
  - `podcast_profile`, `company_profile`, `production_signals`
  - `has_video`
  - `domain`
- POST to OPENROUTER_URL with same pattern as deep_enrich.py:
  - Headers: Authorization Bearer from OPENROUTER_API_KEY env var, Content-Type application/json
  - Body: model from CONFIG, temperature from CONFIG, response_format json_object, messages with system prompt from CONFIG and user message
  - timeout=45 (slightly longer than deep_enrich's 30 since output is more complex)
- Parse JSON response into CallContextResponse
- Return None on any exception (graceful failure per deep_enrich pattern)

**`_generate_one(prospect: ProspectDict) -> ProspectDict`:**
- Call `call_llm_context(prospect)`
- If successful: set `prospect["call_context"]` to `json.dumps(result.model_dump(), ensure_ascii=False)`
- If None: set `prospect["call_context"]` to None
- Return prospect

**`generate_call_context(prospects: list[ProspectDict], workers: int = 4) -> list[ProspectDict]`:**
- Filter to `status == "Qualified"` only (per D-10)
- Separate others (non-Qualified) to pass through unchanged
- Print: `"  Generating call context for {n} Qualified prospects ({m} others skipped, {workers} workers)"`
- Use ThreadPoolExecutor with workers count
- Track done count and failures, print progress per prospect: `"  [{done}/{total}] {podcast_name} - {ok|LLM failed}"`
- Print summary: `"  Call context complete: {n} processed, {f} LLM failures"`
- Return enriched + others

**`if __name__ == "__main__":` block:**
- Load from `data/phone_enrich.json` (or `data/deep_enrich.json` as fallback)
- Run generate_call_context on first 2 Qualified prospects
- Print results for inspection
- Save to `data/call_context.json`

**Storage per D-09:** The main function saves to both local JSON (handled by run_pipeline.py save()) and the call_context field in ProspectDict (which upload.py maps to Airtable).
