# Phase 01.1 Plan 01: Deep Enrichment Module Summary

RSS episode extraction, company homepage scraping, and Ollama-powered theme/summary generation for call-ready prospects.

## What Was Built

Two files that form the deep enrichment stage of the pipeline:

1. **`src/pipeline/prompts/deep_enrich.json`** - Prompt config for Ollama (gemma4:e4b, temperature 0). Single combined prompt generates both podcast_themes and company_summary from episode data and homepage text.

2. **`src/pipeline/deep_enrich.py`** - Module with 7 public functions:
   - `extract_episode_details()` - Parses RSS for top 3 episode details and top 10 titles
   - `extract_guest_name()` - Regex-based guest name extraction from episode titles (4 patterns)
   - `clean_description()` - HTML stripping and whitespace normalization
   - `scrape_company_text()` - Homepage meta tag extraction with /about page fallback
   - `call_ollama_deep()` - Combined LLM call for themes + company summary
   - `assemble_deep_cheat_sheet()` - Richer cheat sheet with deep enrichment data
   - `deep_enrich_prospects()` - Main entry point, processes call-ready prospects only

## Key Design Choices

- **Failure isolation**: Scrape failures and LLM failures are counted but never crash the pipeline or disqualify prospects. On LLM failure, raw company text (truncated to 300 chars) is used as fallback.
- **Follows qualify.py pattern**: Config loading, Ollama call structure, Pydantic validation, and error handling all mirror the existing qualify.py module for consistency.
- **Guest name via title regex only**: Research identified itunes_author at the entry level as unreliable. Four regex patterns cover common episode title conventions (with, ft., featuring, pipe separator).
