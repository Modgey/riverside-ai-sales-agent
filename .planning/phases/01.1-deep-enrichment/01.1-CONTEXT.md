# Phase 1.1: Deep Enrichment - Context

**Gathered:** 2026-04-22
**Status:** Ready for planning

<domain>
## Phase Boundary

Add deep personalization data to call-ready prospects: recent episode content (descriptions, guest names, topics), company summaries scraped from their domain, and podcast theme summaries derived via local LLM. This data enables Phase 2 to generate genuinely personalized cold call openers instead of generic pitches. Runs after scoring, only on call-ready prospects. Zero additional API costs (RSS feeds, public web pages, local Ollama).

</domain>

<decisions>
## Implementation Decisions

### Episode Content Pull
- **D-01:** Re-parse RSS feeds at deep enrich time (not cached from discover). Only call-ready prospects (~10-20) need parsing, so the overhead is trivial. Keeps discover.py focused on its job.
- **D-02:** Extract last 3 episodes with: title, description (Claude picks truncation length), and guest name.
- **D-03:** Guest name extraction uses both sources: itunes:author per entry first, fallback to title parsing heuristics (patterns like "with [Name]", "ft. [Name]", "featuring [Name]", "| [Name]").

### Company About Scrape
- **D-04:** Layered scraping approach. First grab meta description and og:description tags (fast, clean). Then also extract additional context from the page body (hero text, key paragraphs) for richer personalization.
- **D-05:** Claude decides the right depth for page content extraction (homepage only vs. also /about) based on what produces the best personalization data for cold calls.
- **D-06:** Graceful failure handling. 5-10s timeout, catch errors, leave company_summary blank. No retries. Log failure count at end. Prospect remains call-ready without it.

### Podcast Theme Analysis
- **D-07:** Local LLM via Ollama for theme summary. Feed last 10 episode titles+descriptions, get back a 2-3 sentence theme summary. Zero API cost. Already have Ollama running for qualify step.
- **D-08:** Company summary also LLM-refined via Ollama. Pass raw scraped company text through Ollama to produce a clean 2-3 sentence summary. Handles messy HTML artifacts and inconsistent formatting.
- **D-09:** Follow the same Ollama + Pydantic structured response pattern established in qualify.py for consistency.

### Data Flow and Storage
- **D-10:** Merge deep enrichment fields directly into prospect dicts. deep_enrich.json contains the full prospect list (same pattern as score.json). Upload step reads deep_enrich.json instead of score.json.
- **D-11:** Cheat sheet rebuilt after deep enrich. Score.py keeps building basic cheat sheets for all prospects (preserves full-funnel visibility from Phase 1 D-11). Deep enrich rebuilds richer cheat sheets for call-ready prospects only. Two builds, intentionally decoupled.
- **D-12:** New fields added to call-ready prospect dicts: episode_details (list of 3 episode objects), company_summary (string), podcast_themes (string).

### Claude's Discretion
- Episode description truncation length (balance between context richness and data size)
- Company page scraping depth (homepage only vs. homepage + /about, based on what yields best personalization)
- Whether to batch both theme summary and company refinement into a single Ollama call per prospect or keep them separate
- Exact Ollama model choice (should match or be compatible with the model used in qualify)

</decisions>
